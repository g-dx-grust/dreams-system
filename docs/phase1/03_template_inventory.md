## 変換方針

### doc/xls → docx/xlsx は **ローカル Mac で前処理**

**本番ランタイム（Vercel）では `.doc` / `.xls` を変換しない**（LibreOffice は使わない）。運用フローは以下。

1. 旧 `.doc` / `.xls` の様式原本は、あなたのローカル Mac 上で **Word / Excel（または LibreOffice デスクトップ版）で開き、`.docx` / `.xlsx` として名前を付けて保存**
2. 変換後に以下を目視確認
   - レイアウト崩れ（表のセル幅・行高さ）
   - フォント置換（MS明朝 → 代替フォント）
   - 図・画像の位置（印鑑欄・ロゴ）
   - ページ数一致
3. 管理者ユーザーでログインし、テンプレート管理画面から `.docx` / `.xlsx` を Supabase Storage（`templates` バケット）にアップロード
4. アップロード先のパス：`templates/{category_slug}/{template_id}_v{version}.{docx|xlsx}`

### 補助スクリプト（任意）

ローカル環境に LibreOffice が入っているなら、一括変換スクリプトを `scripts/convert-legacy-templates.sh` として管理者が個人で用意しても構わない（リポジトリには含めない or `scripts/` に置くが CI/CD からは呼ばない）。

## 差し込み方式

詳細な実装は `phase3/07_transfer_engine.md` を参照。ここでは**テンプレート作成者が知るべき要約**のみ記す。

### Word（.docx）— docxtemplater（波括弧 1 つ `{ }`）

```
{applicant.name}              ← 単純な値の差し込み
{parcels[0].chiban}           ← 配列アクセス
{today}                       ← 生成日（令和YY年MM月DD日）

{#parcels}
{locationFull} {chimoku} {area}㎡   ← 繰り返し（複数筆）
{/parcels}
```

**テンプレート作成手順：**

1. `.docx` を Word で開く
2. 差し込みたい箇所に `{フィールドパス}` を直接入力（**波括弧 1 つ**。`{{ }}` は使わない）
3. 繰り返し表（例：複数筆）は `{#parcels}` … `{/parcels}` でセクションを囲む
4. **ハイライト対象にしたいプレースホルダーは、該当 Run（その `{...}` を含む文字範囲）を黄色ハイライト済みにしておく**（Word のハイライト機能 → 黄色）。これで生成後も黄色が残る（07_transfer_engine.md の「ハイライト方式 3-A」参照）
5. ファイルを保存
6. テンプレート管理画面で Supabase Storage にアップロード → マッピング設定

### Excel（.xlsx）— exceljs（セル座標 or 名前定義）

**方式 A：セル座標指定（シンプル）**

マッピング設定でプレースホルダー欄に `B5` を入れ、フィールドパスに `applicant.name` を指定する。複数シートにまたがる場合は `Sheet1!B5`。

**方式 B：名前定義（推奨）**

Excel の「名前の定義」で `applicant_name` → `Sheet1!$B$5` のように登録しておく。マッピングのプレースホルダーに `applicant_name` と入れれば、レイアウト変更に強い。

**方式 C：繰り返し行（複数筆）**

繰り返しの開始行・終了行に `loop_parcels_start` / `loop_parcels_end` の名前定義を置く。エンジンが `context.parcels` の要素数だけ行を複製する（Phase 3 後半で実装予定）。

> テンプレ作成時、`.doc` / `.xls` の拡張子のままアップロードするとエラーになる（前処理必須）。
