# Phase 2-3: 案件マスタ機能仕様

## 概要

案件マスタは業務の中心となるエンティティです。1件の申請業務を1案件として管理し、関係者・土地・金額・帳票生成履歴をすべて案件に紐付けます。

---

## 案件種別と対応カテゴリ

| 案件種別 | コード | 対応様式カテゴリ | 主な提出先 |
|---------|--------|---------------|----------|
| 農地転用許可 | `farmland_conversion` | 農地転用許可 | 農業委員会 |
| 土地改良区 | `land_improvement` | 土地改良区 | 豊川総合用水・豊橋南部 土地改良区 |
| 境界確定測量 | `boundary_survey` | 境界確定測量 | 豊川市・豊橋市 |
| 建築許可 | `building_permit` | 建築許可 | 市区町村建築課 |
| その他 | `other` | — | — |

---

## ステータス遷移

```
問い合わせ → 進行中 → 提出済み → 承認済み → 完了
                ↓
             取消
```

| ステータス | コード | 説明 |
|-----------|--------|------|
| 問い合わせ | `inquiry` | 受付・見積前 |
| 進行中 | `in_progress` | 書類作成・申請準備中 |
| 提出済み | `submitted` | 申請書類を提出済み |
| 承認済み | `approved` | 許可・承認が下りた |
| 完了 | `completed` | 入金確認・業務完了 |
| 取消 | `cancelled` | キャンセル・取下げ |

## 画面仕様

### 案件一覧画面

**レイアウト：** テーブル形式（デフォルト）/ カンバン形式（ステータス別）

**表示カラム：**
- 案件番号
- 案件名
- 種別（バッジ）
- ステータス（カラーバッジ）
- 担当者
- 提出先
- 締切日（期限超過は赤字）
- 操作（詳細・編集）

**フィルタ・検索：**
- フリーワード（案件番号・案件名）
- 案件種別
- ステータス
- 担当者
- 締切日範囲
- 期限超過のみ表示（チェックボックス）

**ソート：** 締切日（近い順）、更新日（新しい順）、案件番号

### 案件詳細画面（タブ構成）

#### タブ1: 基本情報

| 項目 | 入力形式 | 必須 |
|------|---------|------|
| 案件名 | テキスト | ○ |
| 案件種別 | セレクト | ○ |
| ステータス | セレクト | ○ |
| 担当者 | ユーザー選択 | - |
| 提出先 | テキスト | - |
| 提出日 | 日付 | - |
| 締切日 | 日付 | - |
| メモ | テキストエリア | - |

#### タブ2: 関係者

**表示項目：**
- 役割（申請者/代理人/隣地所有者など）
- 氏名（スナップショット値）
- 住所（スナップショット値）
- 電話番号（スナップショット値）
- マスタ同期状態（最新/差異あり）
- 操作（編集・再同期・削除）

**操作：**
- 「関係者を追加」→ 人マスタ検索モーダル
- 「スナップショットを手動編集」→ インライン編集
- 「マスタから再同期」→ 確認ダイアログ後に実行

**役割の選択肢：**

| 役割 | コード | 説明 |
|------|--------|------|
| 申請者 | `applicant` | 申請書の名義人 |
| 譲受人 | `transferee` | 農地転用等で土地を取得する人 |
| 譲渡人 | `transferor` | 農地転用等で土地を手放す人 |
| 代理人/行政書士 | `agent` | 申請代理人 |
| 請求先 | `billing` | 報酬の請求先 |
| 隣地所有者 | `neighbor` | 境界確定等で立会が必要な隣地の所有者 |
| その他 | `other` | |

#### タブ3: 土地情報（複数筆対応）

**表示項目（1筆ごと）：**
- 所在（都道府県・市区町村・大字・地番）
- 地目
- 地積（㎡）
- 転用面積（㎡）

**操作：**
- 「筆を追加」「筆を削除」「並び替え（ドラッグ）」

**合計行：** 地積合計・転用面積合計を自動計算して表示

#### タブ4: 金額情報

| 項目 | 説明 |
|------|------|
| 見積金額（税抜） | 数値入力 |
| 消費税率 | 10% / 8% / 0% |
| 見積金額（税込） | 自動計算 |
| 請求金額（税抜） | 数値入力 |
| 請求金額（税込） | 自動計算 |
| 入金金額 | 数値入力 |
| 入金日 | 日付 |
| 未収金 | 自動計算（請求税込 - 入金金額） |
| メモ | テキストエリア |

#### タブ5: 帳票生成

**機能：**
1. 様式を選択（案件種別に応じてフィルタリング）
2. 転記前チェック結果を表示（埋まるフィールド数・不足フィールド）
3. ハイライト ON/OFF
4. 「帳票を生成」ボタン
5. 生成後はダウンロードリンクを表示

**転記前チェック表示：**
```
✅ 転記可能: 10 / 12 フィールド
⚠️ 不足（任意）: billing.address_full
❌ 不足（必須）: agent.name  ← 赤字で警告
```

#### タブ6: 生成履歴

**表示項目：**
- ファイル名
- バージョン
- 生成日時
- 生成者
- ダウンロードボタン

## 案件番号の自動採番

採番は Postgres 側の関数 `public.next_case_number(case_type)` に集約する（02_db_schema.md 参照）。Server Action では RPC を呼んで取得した値を `cases.case_number` に INSERT する。

```ts
// src/server/cases.ts（抜粋）
export async function createCase(input: CaseCreateInput) {
  const user = await requireActiveUser();
  const supabase = await createClient();

  const { data: nextNumber } = await supabase
    .rpc("next_case_number", { p_case_type: input.case_type })
    .single();

  const { data: row, error } = await supabase
    .from("cases")
    .insert({
      case_number: nextNumber,
      case_name: input.case_name,
      case_type: input.case_type,
      status: "inquiry",
      assigned_user_id: input.assigned_user_id ?? null,
      submission_target: input.submission_target ?? null,
      deadline_date: input.deadline_date ?? null,
      memo: input.memo ?? null,
    })
    .select("id, case_number")
    .single();

  if (error) return { ok: false as const, error: "登録に失敗しました" };
  await logAudit({ userId: user.id, action: "case.create", entityType: "case", entityId: row.id, detail: { after: input } });
  return { ok: true as const, data: row };
}
```

案件種別コード対応：

| 案件種別 | コード |
|---|---|
| 土地改良区 `land_improvement` | `LI` |
| 境界確定測量 `boundary_survey` | `BS` |
| 建築許可 `building_permit` | `BP` |
| 農地転用許可 `farmland_conversion` | `FC` |
| その他 `other` | `OT` |

> 採番を DB 関数に寄せた理由：アプリ側で「最大+1」を取ると競合（同時登録）で同一番号が発生しうる。DB 内で単一トランザクション内に収めることで競合リスクを低減する。

## 案件種別ごとの必須様式セット（提案機能）

案件種別を選択すると、「よく使われる様式セット」を自動提案します。

| 案件種別 | 提案様式 |
|---------|---------|
| 農地転用許可 | 5条許可申請書、5条許可委任状、事業計画書、農転4条・5条別紙 |
| 土地改良区 | 農地転用等の通知書、地区除外申請書、誓約書（転用組合員） |
| 境界確定測量 | 公共用地境界確定申請書、立会委任状 |
| 建築許可 | 申請書、理由書、事業計画書 |

この提案はあくまでヒントであり、ユーザーが自由に変更できます。
